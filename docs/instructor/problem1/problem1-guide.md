## よくあるAI生成テストの問題

**パターン1: 存在確認のみ**
```java
assertNotNull(fee);
assertTrue(fee.amount() > 0);  // 50円でも999円でもパス
```

**パターン2: 範囲確認のみ**
```java
assertTrue(fee.amount() <= 1000);  // 800円でも500円でもパス
```

**パターン3: 例外の型のみ**
```java
assertThrows(IllegalArgumentException.class, ...);  // メッセージ未検証
```

**改善後（目指す形）:**
```java
assertEquals(250, fee.amount());  // PBIの計算例を具体的に検証
assertEquals(1000, fee.amount());  // 上限値を具体的に検証
assertTrue(exception.getMessage().contains("返却日"));  // メッセージも検証
```

## キーメッセージ

**カバレッジ vs 品質**
- カバレッジ100% ≠ テストが機能している
- AIは「行を実行するテスト」を書けるが、「ビジネスルールを検証するテスト」を書くとは限らない

**ミューテーションテストの価値**
- テストの品質を可視化する強力な手法
- 「存在確認」→「値の検証」で検知率30% → 90%

**AI時代の開発者スキル**
- AIが生成したテストの品質を評価・改善できるのは開発者だけ
- テストの本質――何を検証すべきか――を理解するスキルがより重要に
