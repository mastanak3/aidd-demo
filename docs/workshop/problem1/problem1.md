## PBI

### 延滞料金計算機能の追加

**ユーザーストーリー:**

> 図書館の利用者として、返却が遅れた場合に延滞料金を知りたい。そうすることで、支払うべき金額を把握できる。

**Acceptance Criteria:**

| # | 条件 | 詳細 |
|---|------|------|
| AC-1 | 一般会員の延滞料金 | 延滞1日あたり50円 |
| AC-2 | プレミアム会員の延滞料金 | 延滞1日あたり20円 |
| AC-3 | 一般会員の上限金額 | 延滞料金の上限は1,000円 |
| AC-4 | プレミアム会員の上限金額 | 延滞料金の上限は500円 |
| AC-5 | 期限内返却 | 貸出期限当日までに返却した場合、延滞料金は0円 |
| AC-6 | 延滞日数の計算 | 貸出期限の翌日から返却日までの日数を延滞日数とする |
| AC-7 | 返却時の表示 | 返却処理時に延滞料金を計算し、結果を返却レスポンスに含める |
| AC-8 | 貸出時の会員種別 | 延滞料金は**貸出時点の会員種別**に基づいて計算する（返却時に会員種別が変更されていても、貸出時の種別を適用する） |

**計算例:**

| 会員種別 | 貸出期限 | 返却日 | 延滞日数 | 計算式 | 延滞料金 |
|----------|----------|--------|----------|--------|----------|
| 一般 | 5/1 | 5/1 | 0日 | - | 0円 |
| 一般 | 5/1 | 5/6 | 5日 | 5 × 50 | 250円 |
| 一般 | 5/1 | 6/10 | 40日 | 40 × 50 = 2,000 → 上限 | 1,000円 |
| プレミアム | 5/1 | 5/11 | 10日 | 10 × 20 | 200円 |
| プレミアム | 5/1 | 7/1 | 61日 | 61 × 20 = 1,220 → 上限 | 500円 |

#### 1-3. テストの実行と修正

生成されたテストを実行してみましょう。**domainパッケージ全体**をテスト対象にします（既存のテストと新規作成したテストの両方が実行されます）。

```bash

```

> **チェックポイント:**
> - テストはすべてパスしましたか？
> - もしエラーが出た場合、どんなエラーでしたか?

AIが生成したテストは、一発でパスしないこともあります。以下のような問題が起きる可能性があります：

- テストコードにコンパイルエラーがある
- テストデータのセットアップが不完全
- 期待値の設定が間違っている
- モックやスタブの設定が不足している

**エラーが出た場合は、AIに修正を依頼してください。**

> **プロンプト例:**
> ```
> テストを実行したところ、以下のエラーが出ました。修正してください。
>

> [ここに実際のエラーメッセージをコピー&ペースト]
>

> ```

すべてのテストがパスするまで、繰り返し修正してください。

> **学びのポイント:**
> - AIが一発で正しいコードを生成できるとは限らない
> - エラーが出た場合は、エラーメッセージを読み、AIに適切なフィードバックを与えることが重要
> - テストコードも実装コードと同様に、反復的な改善が必要

#### 1-4. カバレッジの測定と確認

すべてのテストがパスしたら、カバレッジを測定してみましょう。

```bash
mvn test jacoco:report
```

カバレッジレポートを確認します。以下の**いずれかの方法**で確認してください。

**方法A: ブラウザで確認する場合**

別のターミナルを開いて、HTTPサーバーを起動してください。

```bash
cd target/site/jacoco
python3 -m http.server 38080
```

VS Codeの右下に「ポートが開かれました」という通知が表示されるので、**「ブラウザで開く」**をクリックしてください。
（通知が消えてしまった場合は、VS Codeの「ポート」タブから38080ポートの「ブラウザで開く」アイコンをクリック）

ブラウザでカバレッジレポートが表示されたら、`com.example.library.domain` パッケージのカバレッジ（Cov.列）を確認してください。

**方法B: CLIで確認する場合**

```bash
# domainパッケージのカバレッジを表示
# domainパッケージのカバレッジを表示（1行目: Instruction, 2行目: Branch）
grep -oP 'ctr2">\K\d+%' target/site/jacoco/com.example.library.domain/index.html | head -2
```

1行目がInstruction Coverage、2行目がBranch Coverageです。

> **チェックポイント:**
> - domainパッケージ全体のカバレッジは何%ですか？
> - カバレッジが90%以上ありますか？

**domainパッケージ全体のカバレッジが90%未満の場合**は、AIに追加のテストを書いてもらい、90%以上を目指しましょう。

> **プロンプト例:**
> ```

> 
> カバーされていないコード:
> [カバレッジレポートで赤く表示されている箇所や、カバーされていないメソッド名を記載]
> ```
> 
> ※ XX%は実際のカバレッジ数値に置き換えてください

追加後、再度カバレッジを測定してください。

```bash
mvn test jacoco:report
```

カバレッジレポートを再確認します。先ほどと同じ方法（ブラウザまたはCLI）で確認してください。

ブラウザの場合はHTTPサーバーを再起動してください。CLIの場合は以下を実行してください。

```bash
# domainパッケージのカバレッジを表示（1行目: Instruction, 2行目: Branch）
```

domainパッケージ全体のカバレッジが90%以上になったことを確認してください。ブラウザで確認した場合は、確認後にHTTPサーバーを停止してください（`Ctrl+C`）。

> **チェックポイント:**
