## PBI

### 貸出料金計算機能

**ユーザーストーリー:**

> 図書館の職員として、書籍の貸出時に会員種別に応じた貸出料金を自動計算したい。そうすることで、料金の計算ミスを防ぎ、会員種別ごとに適切な料金を適用できる。

**背景:**

図書館では有料の貸出サービスを導入します。会員種別（一般・プレミアム）によって日額料金が異なります。さらに、新刊書籍には割増料金が適用されますが、プレミアム会員は新刊の割増が免除される特典があります。また、料金には会員種別ごとの上限額が設定されています。

**Acceptance Criteria:**

| # | 条件 | 詳細 |
|---|------|------|
| AC-1 | 会員種別ごとの日額計算 | 一般100円/日、プレミアム80円/日で基本料金を計算できる |
| AC-2 | 新刊加算 | 新刊書籍は基本料金の50%が加算される（ただしプレミアム会員は免除） |
| AC-3 | 上限額 | 会員種別ごとの上限額を超えない（一般1,000円、プレミアム800円） |

---

## 料金計算ルール

以下のルールを**この順番で**適用して料金を計算します。

### ルール1: 基本料金

**基本料金 = 日額 × 貸出日数**

| 会員種別 | 日額 |
|---------|------|
| GENERAL（一般） | 100円 |
| PREMIUM（プレミアム） | 80円 |

### ルール2: 新刊加算

新刊書籍の場合、ここまでの料金に **50%を加算** します（端数切り捨て）。

**ただし、プレミアム会員は新刊加算が免除されます（会員特典）。**

```
if (新刊 かつ プレミアム会員ではない) {
    fee = (int)(fee × 1.5)
}
```

### ルール3: 上限額の適用

料金が上限額を超える場合、上限額に切り詰めます。

| 会員種別 | 上限額 |
|---------|-------|
| GENERAL（一般） | 1,000円 |
| PREMIUM（プレミアム） | 800円 |

### 計算例

**例1:** 一般会員が通常書籍を3日借りる場合

```
基本: 100 × 3 = 300円
新刊: 対象外
上限: 300 ≤ 1,000
→ 300円
```

**例2:** プレミアム会員が新刊を3日借りる場合

```
基本: 80 × 3 = 240円
新刊: プレミアム会員のため免除 → 240円
上限: 240 ≤ 800
→ 240円（※ 一般会員なら 450円になるところ）
```

**例3:** 一般会員が新刊を10日借りる場合

```
基本: 100 × 10 = 1,000円
新刊: 1,000 × 1.5 = 1,500円
上限: 1,500 > 1,000 → 1,000円
→ 1,000円
```

## 手順

この課題では、TDDの **Red → Green → Refactor** サイクルを3ラウンド繰り返します。
各ラウンドで1つずつルールを追加していきます。
